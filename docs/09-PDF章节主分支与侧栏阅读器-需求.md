# PDF 章节主分支与侧栏阅读器 — 产品需求

本文记录 **「上传 PDF 后以章节目录为主干节点，右侧内嵌阅读器，选区可新建对话并反映到左侧树」** 的需求。与「从消息处分叉」的通用交互可对照 [04-新建分支交互需求.md](./04-新建分支交互需求.md)。

---

## 1. 需求概述

| 能力 | 说明 |
| --- | --- |
| **上传 PDF 后的主分支** | 用户上传 PDF 后，将文档的**各章节**解析为**可选取的节点**，在左侧树状视图中作为**主分支/主干**展示。 |
| **从章节追问** | 用户可在树中**点击任意章节节点**，从该节点开始或接续**针对该章节的追问**（对话上下文以该章为锚点）。 |
| **右侧 PDF 阅读器** | 主界面在**浏览器右侧**提供 **PDF 阅读器**区域，与左侧树、中间对话区**同时可见**（具体栅格与断点以设计为准）。 |
| **划选新建对话** | 在阅读器内**选中某段文字**可**新建一条对话**；新对话对应的**节点**须**加入左侧树**，用于**可视化阅读过程与对话分叉**。 |

---

## 2. 用户场景

1. 用户上传 PDF → 系统生成**章节节点树**（主分支）。
2. 用户在左侧**点击某章** → 以该章为入口进行问答或继续多轮对话。
3. 用户在右侧阅读到某段 → **划选** → **新建对话** → 左侧树出现**新节点**，表示从**阅读/选区**分叉出的对话。

---

## 3. 范围与非目标（初稿）

- **范围内**：单 PDF 场景下的章节树、章节点与对话的关联、侧栏阅读器、选区新建对话、树与对话状态同步。
- **可后置**：多 PDF 切换、跨文档聚合、扫描件 OCR、复杂公式的结构识别与章节边界的极端边界情况。

---

## 4. 待澄清（产品 / 技术）

- **章节来源**：PDF 目录/书签、启发式分章、或**用户确认后再定稿**等策略的选择与回退方案。
- **新对话节点的父级**：挂到**当前章节点**下、**上一条同章对话**、或**带页码/摘录的选区元数据**节点等，需在数据模型中明确。
- **布局**：左 / 中 / 右三栏的**最小宽度、折叠、小屏**行为。
