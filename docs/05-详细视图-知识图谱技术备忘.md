# 详细视图（知识图谱式）— 需求与技术栈备忘

> 记录日期：2026-04-24。与左侧「分支缩略图」分工：缩略图以确定性树形导航为主；详细视图面向大图探索，可采用力导向 + Canvas/WebGL。

## 背景与目标

- **现状**：左侧为窄栏分支缩略图（`BranchVisualizer`），适合层次清晰、点击切换分支。
- **规划**：后续可增加**全屏或独立面板的「详细视图」**，在更大画布上探索多分支关系，交互形态接近 Obsidian Graph 等力导向知识图谱（与缩略图互补，不替代缩略图）。

## 功能需求（草案）

- **力导向布局**：节点随模拟展开并趋于稳定；提供可调的力学参数（链接力、斥力、中心力、衰减等），对应界面上的「力度」类控制。
- **交互**：缩放、平移；按需支持节点拖拽；悬停/点击、路径或邻居高亮；可选按角色、`branch_label` 等筛选。
- **数据**：由当前会话的 `ChatNode` 树导出为 `nodes` + `links`（或等价结构），与现有 store / WebSocket 增量更新衔接。
- **入口与模式**：从缩略图或工具栏进入详细视图；可选「仅当前分支子图」与「全图」切换。

## 推荐技术栈

| 层级 | 选型方向 | 说明 |
|------|----------|------|
| 物理/布局 | **d3-force** | `forceLink`、`forceManyBody`、`forceCenter` 等，行业常见 |
| 渲染（中大规模） | **HTML5 Canvas** | 如 `force-graph`：Canvas + d3-force，接入成本较低 |
| 渲染（更大规模） | **WebGL** | 如 **Sigma.js** + **graphology**，或 **Pixi.js** 自绘 |
| 图模型/算法（可选） | **graphology** | 图结构、指标与部分布局能力 |
| 应用层 | 现有 **React** 栈 | 数据来自 zustand 等；避免「一节点一个 DOM」撑大图 |

## 与缩略图的边界

- **缩略图**：以时间/主线可读、窄栏、快速导航为主；**不以**全程力导向为主方案。
- **详细视图**：采用「d3-force 类力导向 + Canvas 或 WebGL 批量绘制 + 缩放/拖拽/参数面板」更合适。

## 后续待细化

- 详细视图的数据范围：单会话整树 / 是否扩展多会话。
- 节点与边的视觉规范：颜色映射（`role`、`branch_label`、当前路径高亮）。
- 性能与规模目标：据此在 Canvas 与 WebGL 之间做最终选型。

## 参考讨论（内部）

- 左侧缩略图优化：优先层次布局（如层式树、防重叠边），而非将力导向用于缩略图主方案。
