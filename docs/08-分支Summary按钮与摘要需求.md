# 分支 Summary 按钮与分支摘要 — 产品需求

本文记录 **「对话/工作流树中，每个分支起点提供 Summary 入口，一键生成分支级一段话摘要」** 的需求。摘要与记忆的一般工程原则（异步、主路径不阻塞等）见 [06-summary与memory.md](./06-summary与memory.md)。

---

## 1. 需求概述

- **目标**：用户在树形视图中，可对 **某一个分支** 一键请求 **该分支全部内容的概括**，得到 **一段话、简洁** 的摘要，并在 UI 上与该分支 **明确关联** 地展示。
- **与节点悬停梗概的区别**：悬停梗概偏 **单节点/认路**（见 [07-对话树节点悬停梗概-需求与方案.md](./07-对话树节点悬停梗概-需求与方案.md)）；本需求是 **整条分支** 的 **主动触发** 的 **段落级** 总结。

---

## 2. 视觉与布局

| 项 | 说明 |
| --- | --- |
| **配色** | **灰色**（非红色）：向上的连线、圆形按钮的描边/填充等与现有主路径、彩色分支区分时，统一采用 **中性灰** 系，避免与错误/警告语义混淆。 |
| **连线** | 自 **分支起点**（从主干分叉处）向上延伸的 **竖向线段**，示意 Summary 控件挂在该分支 **顶端一侧**。 |
| **端点控件** | 线段顶端为 **圆形可点击按钮**，内为字母 **「S」**，表示 **Summary**。 |
| **范围** | **每一个分支** 各自具备上述 **连线 + S 按钮**（非全局唯一入口）。 |

具体色值、线宽、与节点间距可在视觉稿/实现阶段与整体设计系统对齐，需求层面锁定为 **灰色系 Summary 控件**。

---

## 3. 交互

- 用户 **单击** 该分支上的 **S** 按钮后，触发 **分支摘要** 流程。
- 加载中、失败、重试等状态需在实现阶段定义（建议：加载态不打断整棵树操作；失败可提示并允许重试）。

---

## 4. 数据与后端

- **上下文范围**：仅包含 **当前分支** 上的全部节点内容（如用户/助手消息等节点中的文本或约定好的结构化字段），组装为调用大模型所需的 **context**。
- **调用**：将 context 发送至 **大模型 API**，由模型输出 **单段、简洁** 的自然语言（一段话概括该分支讨论主题、过程要点或结论倾向等，具体提示词由实现定义）。
- **工程建议**（与 `06` 一致）：用户点击后的请求可作为 **显式触发的异步操作**，避免阻塞其他 UI；若后续有「自动预生成」，宜走后台任务与持久化，读路径优先读已生成结果。

---

## 5. 展示

- 摘要返回后，须在 **UI 上与该分支绑定展示**（具体形态：侧栏、浮层、分支旁固定区域等由详细设计确定）。
- **非目标**：不强制替代右侧详细视图；摘要为 **该分支的压缩阅读** 辅助。
