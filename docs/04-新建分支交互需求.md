# 新建分支交互需求

## 背景

在 ChatTree 对话界面中，用户需要从某条历史消息处分叉出新对话分支。早期方案依赖「选中消息内文字 + 右键菜单」，操作成本较高。本需求改为基于**鼠标悬浮**的轻量交互，并在分叉后于左侧缩略图（分支树）中展示新分支。

## 目标

- 无需选中文字即可发起「从该条消息创建新分支」。
- 交互路径短：悬浮 → 按钮 → 内联输入 → 发送即创建分支。
- 创建成功后，左侧分支可视化区域能反映新分支结构；当前对话视图可切换到新分支以便继续聊。

## 用户与范围

- **适用消息**：任意一条**用户消息**或**助手消息**（整条消息气泡作为可交互区域）。
- **不适用**：空会话、无消息列表时的占位状态仅展示引导文案。

## 交互流程

### 1. 悬浮

- 当鼠标指针进入某条消息的气泡及其附属区域（含时间戳，避免从气泡移向时间戳时误触「离开」导致按钮闪烁消失）时，视为「悬停在该条消息上」。
- 悬停时，在该消息气泡**正下方**展示一个按钮组件（例如文案「新建分支」，可配图标）。

### 2. 点击按钮

- 在**按钮所在位置**将按钮替换为**内联输入区**（输入框 + 发送）。
- 输入内容表示新分支的**首条用户消息**（与主输入区发送新消息的语义一致，但 `parent_node_id` 为被悬停的那条消息对应节点）。

### 3. 输入与发送

- 用户在内联输入框中输入文字，通过「发送」按钮或 Enter（单行场景）提交。
- 发送后：向后端 / WebSocket 发送聊天请求，携带：
  - `parent_node_id`：被分叉的那条消息节点 ID；
  - `message`：首条用户消息正文；
  - `branch_label`（可选）：用于左侧缩略图/节点上的短标签，可由首条消息截断生成（如前 48 字）。

### 4. 取消

- 用户可通过 `Esc` 或点击输入区域外关闭内联输入，不创建分支。

### 5. 创建完成后的界面

- **左侧缩略图 / 分支树**：新分支作为从父节点伸出的子分支展示；若节点支持 `branch_label`，应显示该标签以便区分。
- **主对话区**：创建成功后，宜将当前视图**切换到新分支**（以新分支上的用户消息为路径终点），使用户继续对话时自然走新分支；若实现上采用「仅刷新树、不自动切换」，需在文档或界面中说明差异（当前实现为自动切换）。

## 非目标 / 已废弃

- **不再要求**：必须先选中消息内文字再通过右键菜单创建分支。
- **未纳入本需求**（除非产品另行规定）：仅当鼠标悬停在英文某几个字母（如 `u` / `a`）上才出现按钮；若需字符级命中，应单独开需求说明规则与多语言场景。

## 技术要点（实现参考）

- 前端：`ChatView` 维护悬停消息 ID、内联分叉输入状态；`ChatTreeView` 调用 `sendMessage` 并处理新用户节点到达后的当前分支切换与 `pending` 匹配（建议用 `parent_id` + 首条消息内容避免同父多叉竞态）。
- 后端：沿用现有 WebSocket `chat` 消息及 `branch_label` 字段即可。

## 验收要点

1. 不选中任何文字，仅通过悬浮即可在气泡下出现「新建分支」。
2. 点击后出现内联输入，发送后产生新分支节点，左侧树可见新分叉。
3. 发送后主对话区进入新分支上下文，可继续收发消息。
4. Esc / 点击外部可取消内联输入且无请求发出。
