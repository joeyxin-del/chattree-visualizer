# 对话树节点悬停梗概 — 需求与实现方案

本文记录 **「左侧对话树悬停某节点时展示该节点一句话梗概」** 的产品需求，以及**不落代码**层面的实现方案；摘要生成与记忆分层的一般原则见 [06-summary与memory.md](./06-summary与memory.md)。

---

## 1. 需求概述

- **行为**：当鼠标 **悬停** 于左侧**对话树**的某一个**节点**上时，在合适位置（如浮层/Tooltip 形态）**弹出**关于该节点的一句话**简要说明**，帮助用户快速识别「这条（分支/会话）在讲什么」，减少误点。
- **非目标**（不替代）：
  - 不替代右侧**详细视图**或正文；梗概是**扫一眼的辅助信息**。
  - 不保证悬停是**唯一**获取该信息的途径（无鼠标/触屏需另有入口，见 §3.3）。

---

## 2. 目标与边界

| 项 | 说明 |
| --- | --- |
| **内容形态** | 非交互的**短文案**（**一句话**量级；实现时可设最大字数，如 60～120 字，产品可再定）。 |
| **体验** | 低打扰：快速划过时不宜满屏弹层；信息应**可读、可扫完**（与常见 Tooltip/Popover 的篇幅约束一致）。 |
| **平台** | 桌面**悬停**为主力；**触摸/键盘**上需有**等效可发现性**（见 §3.3）。 |
| **数据性质** | 属于 **UI 认路/预览** 的「极浅层摘要」；与 Agent 用于续写的**长记忆、关键事实**可共用数据源，但**字段与刷新策略**上宜分开展开（与 `06` 中 Summary vs 选择性记忆 的分工一致）。 |

---

## 3. 实现方案（架构级）

### 3.1 数据从哪来（推荐：持久化 + 双轨内容）

- **主展示字段**（建议名如 `previewText` / `oneLiner`）：**持久化在服务端**（或与会话/线程绑定的元数据上），供树结构接口或节点详情**直接读取**，保证 **hover 读路径 0 次同步大模型**。
- **内容两轨**：
  1. **规则/截断（默认与降级，始终可用）**  
     如：首条用户消息截断、已有会话标题、父路径标签、时间等，拼成**一句**或固定模板；在冷启动、异步摘要未返回、大模型失败时**兜底**。
  2. **异步大模型摘要（质量增强，可选或分阶段上）**  
     在会话/消息**更新后**由后台任务生成**一句**并**写回** `previewText`；**不阻塞**用户发消息、打开会话等主路径（与 `06` 中「主路径不强依赖现算摘要」一致）。

- **不推荐的唯一依赖**：**仅**在用户悬停时**同步**调大模型现算。可做**可选增强**（带防抖、取消、失败回退到规则轨），但**不得**作为唯一文案来源，否则延迟、成本与划树误触问题突出。

### 3.2 何时写入、写多少

- **触发时机**：消息落库或会话**显著更新**后 enqueue 异步 job；也可采用**阈值策略**（如每 N 轮、token/体量超阈值、距上次摘要超过一定时间再刷新），避免「每条消息都全量重概括」带来的成本（参见 `06` §3、§4）。
- **与批量导入**：允许摘要**晚若干秒到数分钟**就绪；期间 UI 仅用**规则/截断**展示（见 `06` §2）。

### 3.3 读路径与 UI 行为

- **悬停（桌面）**：`mouseenter` 后经 **100～300ms 延迟**再显示浮层，减少快速划树时频繁闪烁；用户指针应能**移入浮层**阅读而不至于立刻消失（满足常见 WCAG 对 hover 浮层可停留的期望，实际实现时选用成熟 Tooltip/Popover 组件并测键盘与读屏行为）。
- **无 hover 场景**：**焦点**到树节点时也可展示**同一段**梗概；**触摸**设备用**长按**、或**选中节点后在侧栏/固定区域**显示副标题/梗概，避免「只有悬停能看」。

### 3.4 接口形态（落实现代码时的约定，本文仅作形状说明）

- 树/会话列表的接口**在节点级**返回 `previewText`（或列表后再 **batch 按 id 拉一批梗概**），注意避免 **N+1** 请求，按现有后端切分选择其一。
- 若**一个会话对应多树节点**的模型未来出现，再单独约定「梗概以会话为维度单点存储、多节点引用」；**当前**若节点与会话 1:1，则**按会话存一份**即可。

### 3.5 与 `06` 的衔接

- **生成策略、异步、降级、每轮全量的成本问题**：以 [06-summary与memory.md](./06-summary与memory.md) 为**理论基线**；本需求是其在 **「左侧树 + 一句话预览」** 上的**产品落地**。
- 若内部另有 **长 summary / 向量记忆 / 知识图谱**，**不必**与 `previewText` 强绑同一字段；可后台任务**一次产出多型产物**，前端树节点**只读最短那一列**。

---

## 4. 验收建议（可转测试清单）

- [ ] 无大模型、无网络异常时，悬停/焦点仍能看到**稳定的一句**（来自规则/截断或标题等）。
- [ ] 大模型异步未就绪时，不空白、不长时间转圈；**有降级**。
- [ ] 在树上**快速划过多节点**，不因悬停对后端或模型造成**可感知的重复风暴**（前端防抖/后端幂等/缓存策略按实现选型）。
- [ ] 批量导入或大量新消息后，梗概在合理时间内**陆续更新**可接受，或由产品规定「仅手点刷新/进入详情后更新」。
- [ ] 至少一种**无鼠标**方式能读到**实质相同的梗概**（或明确产品声明「仅桌面悬停提供」并文档化，一般不推荐）。
