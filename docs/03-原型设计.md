# 树形可视化项目 - 原型设计

## 1. 设计原则

### 1.1 核心原则
- **清晰性**：树形结构一目了然
- **高效性**：快速定位和导航
- **响应性**：适配不同屏幕尺寸
- **一致性**：统一的交互模式

### 1.2 设计目标
- 支持 100+ 节点的流畅展示
- 3 秒内完成任意节点定位
- 移动端可用性 ≥ 80%

## 3. 核心组件设计

### 3.1 节点（Node）设计

#### 3.1.1 用户消息节点
```
┌─────────────────────────────────┐
│ 👤 User                    [+]  │  ← 角色标识 + 分支按钮
├─────────────────────────────────┤
│ 帮我分析一下土耳其旅游的优缺点    │  ← 消息内容
├─────────────────────────────────┤
│ 🕐 2024-03-16 10:30        [3] │  ← 时间戳 + 子节点数
└─────────────────────────────────┘
```

### 3.2 连线（Edge）设计

#### 3.2.1 直线连接（默认）
```
    ┌─────┐
    │Node1│
    └──┬──┘
       │  ← 实线，表示父子关系
    ┌──┴──┐
    │Node2│
    └─────┘
```

#### 3.2.2 曲线连接（美化）
```
    ┌─────┐
    │Node1│
    └──┬──┘
       ╰─┐  ← 贝塞尔曲线
    ┌────┴─┐
    │Node2 │
    └──────┘
```

#### 3.2.3 多分支连接
```
       ┌─────┐
       │Root │
       └──┬──┘
          │
    ┌─────┼─────┐
    │     │     │
 ┌──┴──┐ │  ┌──┴──┐
 │Node1│ │  │Node3│
 └─────┘ │  └─────┘
      ┌──┴──┐
      │Node2│
      └─────┘
```

## 4. 交互设计

### 4.1 节点交互

#### 4.1.1 点击节点
- **单击**：选中节点，显示详情
- **双击**：聚焦节点（居中显示）
- **右键**：显示上下文菜单

#### 4.1.2 上下文菜单
```
┌─────────────────┐
│ 创建分支        │
│ 复制内容        │
│ 删除节点        │
│ ───────────────│
│ 折叠子树        │
│ 展开全部        │
│ ───────────────│
│ 导出为图片      │
└─────────────────┘
```

### 4.2 画布交互

#### 4.2.1 缩放
- **鼠标滚轮**：缩放
- **Ctrl + 滚轮**：更快缩放
- **双指捏合**：移动端缩放

#### 4.2.2 平移
- **鼠标拖拽**：平移画布
- **空格 + 拖拽**：临时平移模式
- **双指拖动**：移动端平移

#### 4.2.3 选择
- **框选**：拖拽选择多个节点
- **Ctrl + 点击**：多选
- **Shift + 点击**：范围选择

### 4.3 键盘快捷键

```
Ctrl + F      搜索节点
Ctrl + Z      撤销
Ctrl + Y      重做
Ctrl + A      全选
Delete        删除选中节点
Ctrl + D      复制节点
Ctrl + +      放大
Ctrl + -      缩小
Ctrl + 0      重置缩放
Space         临时平移模式
Esc           取消选择
```

## 6. 布局模式

### 6.1 垂直布局（默认）

```
        ┌─────┐
        │Root │
        └──┬──┘
           │
    ┌──────┼──────┐
    │      │      │
 ┌──┴──┐┌──┴──┐┌──┴──┐
 │Node1││Node2││Node3│
 └─────┘└─────┘└─────┘
```

**适用场景**：对话树、时间线

### 6.2 水平布局

```
┌─────┐    ┌─────┐    ┌─────┐
│Root ├────┤Node1├────┤Node2│
└─────┘    └──┬──┘    └─────┘
              │
           ┌──┴──┐
           │Node3│
           └─────┘
```

**适用场景**：流程图、决策树

### 6.3 径向布局

```
        ┌─────┐
    ┌───┤Node1│
    │   └─────┘
┌───┴──┐
│ Root │
└───┬──┘
    │   ┌─────┐
    └───┤Node2│
        └─────┘
```

**适用场景**：关系图、思维导图

### 6.4 紧凑布局

```
┌─────┐
│Root │
└──┬──┘
   ├─ Node1
   ├─ Node2
   │  ├─ Node2.1
   │  └─ Node2.2
   └─ Node3
```

**适用场景**：大量节点、移动端

## 7. 响应式设计

### 7.1 断点

```
Mobile:   < 768px
Tablet:   768px - 1024px
Desktop:  > 1024px
```

### 7.2 适配策略

#### 7.2.1 桌面端（> 1024px）
- 三栏布局（侧边栏 + 画布 + 详情面板）
- 完整功能
- 鼠标交互优先

#### 7.2.2 平板端（768px - 1024px）
- 两栏布局（画布 + 可折叠侧边栏）
- 简化工具栏
- 触摸 + 鼠标混合交互

#### 7.2.3 移动端（< 768px）
- 单栏布局（全屏画布）
- 底部工具栏
- 触摸交互优先
- 紧凑布局模式
