## 🚀 立即开始使用

### 1. 打开前端界面

在浏览器中访问：
```
http://localhost:5173
```

### 2. 开始第一个对话

- 在右侧输入框输入你的问题
- 按 Enter 或点击"发送消息"
- 等待 Claude AI 的实时流式回复

### 3. 创建分支对话

- 点击任意对话节点（用户或助手消息）
- 在右侧输入新的问题
- 发送后会在该节点下创建新分支

### 4. 探索树形结构

- **拖拽画布** - 移动视图
- **鼠标滚轮** - 缩放
- **点击节点** - 选中并创建分支
- **小地图** - 快速导航

## 📊 功能演示

### 对话树可视化
```
        [用户问题]
            ↓
        [AI 回答]
        ↙    ↓    ↘
   [分支1] [分支2] [分支3]
```

### 实时流式响应
- AI 回复会实时显示，就像打字一样
- 节点状态实时更新（思考中 → 完成）

### 多分支并行
- 可以同时在多个分支提问
- 每个分支独立处理

## 🔍 API 端点

### REST API
- `GET /` - 根路径，返回 API 信息
- `POST /api/sessions` - 创建新会话
- `GET /api/sessions/{session_key}` - 获取会话数据

### WebSocket
- `WS /ws/{session_key}` - 实时通信
  - 发送消息
  - 接收流式响应
  - 节点状态更新

### API 文档
访问 http://localhost:8000/docs 查看完整的交互式 API 文档

```

  ├─ 分支1：土耳其签证要求？

```
问题：Python 装饰器是什么？
  └─ 回答：装饰器是...

  └─ 角度3：配置优化

## 🎨 界面说明

### 左侧：树形画布
- 显示所有对话节点
- 用线条连接父子关系
- 支持拖拽和缩放

### 右侧：控制面板
- **顶部**: 当前选中节点信息
- **中间**: 消息输入框
- **底部**: 统计信息（节点数、根节点数）

### 节点颜色
- 🔵 **蓝色** - 用户消息
- 🟢 **绿色** - AI 回答
- 🟡 **橙色** - 思考中
- ⚪ **灰色** - 已取消/错误

## 🐛 故障排除

### 前端无法连接后端
1. 确认后端正在运行（已确认 ✓）
2. 打开浏览器开发者工具（F12）
3. 查看 Console 标签的错误信息
4. 查看 Network 标签的请求状态

### WebSocket 连接失败
1. 刷新页面重试
2. 检查防火墙设置
3. 确认端口 8000 未被阻止

### AI 回复错误
1. 检查 API Key 是否正确
2. 查看后端日志
3. 确认网络连接正常
